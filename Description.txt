controllers => ทำหน้าที่เป็นตัว C ใน MVC
-endpoint.js --> ใช้ในการรับคำสั่งมาจาก Router และมีการดึงตัวชุดโครงข้อมูลมาจาก M ที่ models/model.js มาเพื่อส่งคำสั่งและโครงข้อมูลออกจาก endpoint.js ไปที่ logic.js 
-logic.js --> รับคำสั่งมาจาก endpoint.js ที่ logic.js จะเป็นจุดรวมคำสั่งที่ใช้ในการทำตามความต้องการของโจทย์ และส่งชุดคำสั่งที่ผ่านการประมวลผลแล้วไปที่ operation.js
operation.js --> รับคำสั่งหรือข้อมูลมา และทำหน้าที่ในการทำ Event กับตัวของขอมูล เช่นการเขียนคำสั่งเพื่อติดต่อเอาข้อมูลที่มาจาก logic.js ไปติดต่อกับ Database ที่ connection.js

database => ทำหน้าที่เป็นฐานข้อมูล
-connection.js --> สร้างการเชื่อมต่อกับฐานข้อมูลกับฐานข้อมูลที่ใช้

models => ทำหน้าที่เป็นตัว M ใน MVC
-model.js --> สร้างโครงสร้างของข้อมูลในไฟล์นี้เพื่อใช้เป็นโครงสร้างของข้อมูลตามความต้องการของโจทย์

routes => เก็บเส้นทางทั้งหมดที่มี
-router.js --> ใช้เป็นตัวนำทางให้กับ Event ว่าการกระทำนี้จะไปที่ไหนและทำอะไรต่อกับเส้นทางนั้น

views => ทำหน้าที่เป็น V ใน MVC
-edit.ejs --> เป็นหน้าตาที่ใช้ติดต่อกับผู้ใช้งานโดยหน้าที่ของ edit.ejs จะเป็นการแก้ไขข้อมูลที่มีอยู่และส่งอัพเดทไปที่ controllers
-index.ejs --> เป็นหน้าตาที่ใช้ติดต่อกับผู้ใช้งานโดยหน้าที่ของ index.ejs จะเป็นตัวหน้าใช้งานหลักในการเชื่อมกับหน้าอื่นๆได้และในโจทย์นี้ใช้หน้านี้ในการรับข้อมูลจาก User
-navbar.ejs --> เป็นแถบอำนวยความสะดวกในการติดต่อกับหน้าอื่นๆ ใช้เป็นทางลัดในการไปหน้าอื่นๆ
-show_table.ejs -->เป็นหน้าตาที่ใช้ติดต่อกับผู้ใช้งานโดยหน้าที่ของ show_table.ejs จะเป็นตัวหน้าที่แสดงผลข้อมูลที่ดึงมาจากตัว ฐานข้อมูลที่ใช้เก็บและสามารถแก้ไขข้อมูลหรือลบข้อมูลผ่านหน้านี้ได้

app.js --> ใช้ในการเรียก Package ที่ใช้งานทั้งหมด และกำหนดโครงสร้างที่ใช้งานว่าใช้อะไรบ้างจาก Package และจะมีคำสั่งในการเปิด Local Server ผ่านตัวนี้
